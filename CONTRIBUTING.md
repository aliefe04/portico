# Contributing to Portico

Thanks for helping build Portico.

## Principles

- Keep the app trustworthy and transparent
- Prefer small, targeted changes
- Add or update tests when behavior changes
- Avoid introducing secret storage or hidden side effects

## Workflow

1. Branch from `master` for each change
2. Keep PRs focused on one idea
3. Include screenshots or terminal output when useful
4. Make sure tests pass before asking for review
5. Use feature branches, not worktrees

## Code style

- Use idiomatic Go
- Keep package boundaries small
- Favor clear names over clever abstractions
